AppLauncher:
AppLauncher checks the availability of all Application Launcher's UI applications during an upgrade. If "AppLauncher" is marked red on the Availability Report. PLEASE check ApplicationLauncher Report for more details.
1. Missing Links :- A list of links, compiled pre-upgrade from those available on the system, are tested iteratively for availability during the upgrade. The result is stored in the format: number of times link was missing / number of iterations.
2. Failure Requests :- Once it is confirmed that a link is not missing, it is then confirmed that the link returns a 200 response code when clicked. The result is stored in the format: number of iterations where 200 response code was not received / number of iterations
3. Failure Criteria :- If All Application links failed for same iteration during Upgrade, it will display failure count in below table and ADU report will be marked as failed.
